
# MySave 확장프로그램 화면 기획안 v1

> 본 문서는 **MySave Chrome 확장프로그램 저장 팝업 UI**에 대한 화면 구성 및 동작 기획을 정의한다.

---

## 1. 화면 개요

- **대상 플랫폼:** Chrome Extension (Popup)
- **주요 목적:**  
  사용자가 현재 보고 있는 웹페이지를 빠르게 저장하고,  
  메모·태그·리마인드 정보를 함께 등록할 수 있도록 지원한다.

---

## 2. 화면 구성 요소 상세

| NO | 항목 | Description |
| --- | --- | --- |
| **1** | **헤더 (로고)** | a. **MY SAVE** 로고 노출<br> b. 로고 클릭 시 메인 웹 대시보드(홈)로 새 탭 이동 *(선택 사항)* |
| **2** | **기본 정보 입력<br>(URL, 제목, 텍스트)** | a. **초기 진입 시 자동 입력 (Auto-fill)**<br> - 현재 활성 브라우저 탭의 URL, Page Title 자동 입력<br> - 사용자가 드래그(Highlight)한 텍스트가 있을 경우 ‘텍스트’ 영역에 자동 입력<br> b. **수정 가능**: 사용자가 직접 내용 수정 가능 |
| **3** | **요약한 텍스트<br>(AI 요약)** | a. **Placeholder**: “AI가 내용을 요약해줍니다.” 문구 노출<br> b. **요약 로직**<br> - (Case A) 팝업 로딩 시 자동 요약 실행<br> - (Case B) 별도 버튼 없이 텍스트 입력 시 백그라운드 자동 처리 또는 저장 시 처리됨을 명시 |
| **4** | **추가 정보<br>(메모, 태그)** | a. **메모**: 사용자 입력 텍스트 영역 (Multi-line)<br> b. **태그**<br> - 태그 입력 후 Enter 키 입력 시 태그 생성 (Chip 형태)<br> - 생성된 태그 옆 ‘x’ 버튼으로 삭제 가능 |
| **5** | **리마인드 설정<br>(토글 스위치)** | a. **Default**: OFF 상태 (좌측 회색)<br> b. **인터랙션**: 토글 클릭 시 ON / OFF 전환<br> c. **UI 변경**<br> - OFF: 하단 날짜 선택 영역 숨김<br> - ON: 하단 날짜 선택 영역 노출 |
| **5-1** | **리마인드 옵션<br>(빠른 선택)** | *리마인드 ON 상태에서만 활성화*<br> a. 빠른 설정 버튼 3종: `[내일 아침]`, `[이번 주말]`, `[다음 주]`<br> - 클릭 시 Active 스타일 적용<br> - 각 버튼 하단에 실제 요일/시간 표시 (예: 09:00, 토요일) |
| **5-2** | **리마인드 옵션<br>(직접 선택)** | *리마인드 ON 상태에서만 활성화*<br> a. **직접 날짜/시간 선택하기** 버튼 제공<br> b. 클릭 시 Date Picker & Time Picker 팝업 또는 드롭다운으로 사용자 지정 가능 |
| **6** | **저장하기 버튼** | a. **유효성 검사(Validation)**<br> - 필수 항목(URL 등) 미입력 시 저장 불가 및 알림 표시<br> b. **Action**<br> - 모든 입력 데이터(URL, 제목, 텍스트, 요약, 메모, 태그, 리마인드 정보)를 서버로 전송<br> - 저장 성공 시 “저장되었습니다” 토스트 팝업 노출 후 확장 프로그램 창 닫기 |

---

## 3. UX 설계 의도

- 사용자의 **행동 흐름을 최소화**하여 1~2번 클릭만으로 저장 가능
- 저장 시 **맥락 정보(메모 + 요약)**를 함께 남겨 재방문 효율 극대화
- 리마인드 기능을 **선택형 UX**로 제공하여 부담 없이 사용 가능

---

## 4. 참고 사항

- 본 기획안은 **v1 (AI 기능 포함 버전)** 기준이며  
  추후 MVP 범위에 따라 요약 영역 제거 또는 비활성화 가능
- 실제 구현 시 Chrome Extension 보안 정책(CSP, Content Script 제한)을 고려해야 함
