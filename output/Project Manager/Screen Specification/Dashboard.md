# MySave 웹 대시보드 화면 기획안

본 문서는 MySave 웹 서비스의 **대시보드(Dashboard) 화면 구성 및 기능 정의**를 목적으로 한다.  
사용자가 저장한 콘텐츠를 한눈에 파악하고, 다시 읽을 수 있도록 돕는 허브 화면이다.

---

## 📌 대시보드 화면 구성 요소

| **NO** | **항목** | **Description** |
| --- | --- | --- |
| **1** | **GNB (상단 메뉴)** | a. **메뉴 탭**: `[대시보드]`, `[북마크 목록]`, `[태그]`, `[리마인드]`, `[설정]`<br> - 현재 활성 메뉴(대시보드)에 `Active` 스타일 적용 (밑줄 또는 강조 색상)<br><br> b. **검색창**<br> - Placeholder: *"저장한 글, 태그 검색"*<br> - Enter 키 입력 또는 돋보기 아이콘 클릭 시 검색 결과 페이지 이동<br><br> c. **북마크 추가하기 버튼**<br> - 클릭 시 **URL 직접 입력 모달 팝업** 노출 |
| **2** | **상단 통계 카드 (Summary)** | **2-1. 오늘 저장한 글**<br> - 오늘 기준 저장된 북마크 건수 표시 (예: 3건)<br> - 전일 대비 증감 문구 표시 (예: “어제보다 많이 저장했어요”)<br><br> **2-2. 미완료 리마인드**<br> - 오늘 마감 예정 또는 기한이 지난 미완료 리마인드 건수 표시<br><br> **2-3. 가장 자주 쓴 태그**<br> - 최근 7일 기준 사용 빈도 1위 태그 노출 (태그명 + 아이콘)<br><br> **2-4. 주간 읽기 달성률**<br> - (읽음 처리된 콘텐츠 수 / 전체 저장 수) 기준으로 퍼센트 표시 |
| **3** | **최근 저장 목록 (Main Content)** | a. **헤더**: ‘최근 저장 목록’ 타이틀 + `더보기` 버튼<br> - 더보기 클릭 시 전체 북마크 목록 페이지 이동<br><br> b. **카드 리스트**<br> - 최신순 기준 최대 6개 카드 노출 (Grid 형태)<br><br> c. **카드 상세 구성**<br> - **썸네일 / 배경 영역**: 읽음 상태 배지(Chip), 읽음 토글(Toggle) 오버레이<br> - **정보 영역**: 콘텐츠 타이틀(1줄 말줄임), 대표 태그, 저장 날짜(YYYY.MM.DD)<br> - **즐겨찾기**: 우측 하단 별(Star) 아이콘 토글 ON / OFF |
| **4** | **예정된 리마인드 (Side Widget 1)** | a. **헤더**: ‘예정된 리마인드’ 타이틀 + `더보기` 버튼<br><br> b. **그룹핑**<br> - `[오늘]`, `[내일]` 등 날짜 기준 그룹으로 리스트 구성<br><br> c. **리스트 아이템**<br> - 리마인드 시간 (예: 09:00) + 콘텐츠 타이틀 노출<br> - 클릭 시 해당 콘텐츠 상세 페이지 또는 원본 페이지 이동 |
| **5** | **태그 위젯 (Side Widget 2)** | a. **자주 쓰는 태그**<br> - 사용 빈도 상위 태그를 Pill 형태로 노출<br><br> b. **즐겨찾는 태그**<br> - 사용자가 즐겨찾기 설정한 태그를 고정 영역에 노출<br><br> c. **인터랙션**<br> - 태그 클릭 시 해당 태그로 필터링된 검색 결과 페이지 이동 |

---

## 🎯 대시보드 설계 핵심 의도

- 사용자의 **저장 활동과 소비 상태를 한눈에 파악**
- 리마인드 중심의 **재방문 동기 강화**
- 태그 기반의 **자기주도적 정보 정리 경험 제공**
