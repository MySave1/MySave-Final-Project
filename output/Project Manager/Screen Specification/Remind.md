# MySave 리마인드(Remind) 화면 기획안

본 문서는 MySave 서비스의 **리마인드 관리 화면 UI 및 동작 흐름**을 정의한다.  
사용자는 설정한 일정에 따라 다시 읽어야 할 콘텐츠를 시간 순으로 확인할 수 있다.

---

## 📌 리마인드 화면 구성 요소

| **NO** | **항목** | **Description** |
| --- | --- | --- |
| **1** | **GNB (상단 메뉴)** | a. **메뉴 탭**: `[리마인드]` 탭 활성화 (Active Style)<br><br> b. **검색창**<br> - 키워드 입력 시 **리마인드 목록 내 제목 기준 검색 / 필터링** |
| **2** | **알림 배너 (Hero Section)** | a. **동적 텍스트(Counter)**<br> - 문구: *"오늘 마감되는 글 **N건**이 있어요"*<br> - 당일(Today) 날짜로 설정된 리마인드 개수를 계산하여 N에 표시<br><br> b. **배경 스타일**<br> - 강조 색상(Blue 계열) 적용하여 상단 주목도 강화 |
| **3** | **타임라인 컨테이너 (Structure)** | a. **정렬 기준**<br> - 리마인드 설정 시간 기준 **오름차순(가장 임박한 일정 우선)**<br><br> b. **날짜 기준 섹션 그룹핑**<br> - **[🔵 오늘]**: 당일 리마인드 일정<br> - **[🟡 내일]**: 익일 리마인드 일정<br> - **[🔴 이번 주 / 다음 주]**: 그 외 미래 일정<br> *(섹션 타이틀은 날짜 차이에 따라 동적으로 변경 가능)* |
| **4** | **리마인드 카드 (Reminder Card)** | a. **시간 표시 (좌측 영역)**<br> - **오늘 / 내일**: 실제 시간 표시 (예: *오후 2:00*)<br> - **그 외 일정**: D-Day 형식 표시 (예: *D-2*, *D-6*)<br><br> b. **기본 정보**<br> - 썸네일 또는 아이콘(Bookmark Icon)<br> - 콘텐츠 제목(Title)<br> - 원본 URL 도메인만 노출 (예: *medium.com*)<br><br> c. **원본 글 보기 버튼**<br> - 우측 하단 링크 아이콘 + 텍스트 제공<br> - 클릭 시 **새 탭으로 원본 페이지 이동**<br><br> d. **카드 전체 클릭 이벤트**<br> - 카드 영역 클릭 시 **상세 보기 페이지 또는 원본 페이지 이동**<br> - 개발 참고: 버튼과 카드 본문의 이동 경로가 동일할 경우 `<a>` 태그로 카드 전체를 감싸는 구조 권장 |

---

## 🎯 리마인드 화면 설계 핵심 가치

- **시간 흐름 기반(Time-oriented)** 콘텐츠 재소비 경험
- 오늘 해야 할 읽을거리를 한눈에 보여주는 **우선순위 인지**
- 불필요한 조작 없이 바로 원문으로 이동 가능한 **빠른 접근성**
