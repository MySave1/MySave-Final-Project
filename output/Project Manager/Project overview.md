## 2. 기획 배경 (Background & Problem)

### Pain Point (문제점)

1. **북마크의 무덤화**  
   유용한 글을 발견해 저장하지만, 다시 열어보지 않고 쌓여만 감.

2. **저장 맥락의 소실**  
   제목과 URL만 남아 있어, 나중에 *왜* 저장했는지 기억하기 어려움.

3. **체계적인 관리의 부재**  
   기본 브라우저 북마크는 태그, 상태 관리, 리마인드 기능이 부족함.

---

### Solution (해결책)

1. **메모 중심 북마크**  
   저장 시 사용자가 직접 메모를 남겨, 북마크의 목적과 맥락을 기록하도록 유도함.

2. **리마인드 기반 소비 구조**  
   ‘언제 다시 볼지’를 설정하게 하여, 저장으로 끝나지 않고 실제 소비로 이어지도록 설계함.

3. **태그 기반 분류 시스템**  
   사용자가 직접 태그를 관리함으로써, 개인화된 기준으로 북마크를 구조화할 수 있도록 지원함.

---

## 3. 핵심 목표 (Goals)

1. **재방문율 향상**  
   리마인드 알림을 통해 저장된 북마크를 다시 열어보는 비율을 높임.

2. **맥락 회복 시간 단축**  
   저장 당시 작성한 메모와 태그를 통해, 링크의 의도를 빠르게 파악할 수 있도록 함.

3. **지식 관리 습관 형성**  
   단순 링크 저장이 아닌, 읽기·정리·완료까지 이어지는 개인 지식 관리 흐름을 제공함.

---

## 4. 기대 효과 (Expected Outcome)

1. **북마크 활용도 증가**  
   리마인드 기능을 통해 저장된 콘텐츠의 실제 열람률 향상 기대.

2. **정보 관리 스트레스 감소**  
   태그와 상태 관리 기능으로 북마크 정리에 대한 부담 감소.

3. **개인화된 지식 아카이브 구축**  
   사용자의 메모와 분류 기준이 반영된, 장기적으로 활용 가능한 지식 저장소 형성.

---

## 5. 기능 범위 정의 (Scope)

### 포함 기능 (In Scope)

- 북마크 CRUD (생성, 조회, 수정, 삭제)
- 태그 관리 및 태그 기반 필터링
- 리마인드 알림 설정 및 상태 관리
- Chrome Extension 연동 저장

### 제외 기능 (Out of Scope)

- AI 자동 요약 기능
- AI 태그 추천 기능
- 자연어 검색 기능
